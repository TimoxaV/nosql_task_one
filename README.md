### Part 2
1. Для чого використовується інструкція $unwind?
$unwind створює по окремому документу на кожне значення в масиві. Наприклад, трек із масивом виконавців artists: ["Drake", "Dr.Dre"] створить два документи: один з artists: "Drake", а інший - з artists: "Dr.Dre". Це необхідно зробити перед групуванням за окремими виконавцями.

2. Чим $stdDevPop відрізняється від $stdDevSamp?
$stdDevPop - це стандартне відхилення генеральної сукупності (ділення на N) і застосовується,коли згруповані документи є всією генеральною сукупністю (у колекції є абсолютно всі треки певного жанру).

$stdDevSamp - це вибіркове стандартне відхилення (ділення на N-1). Застосовується, коли згруповані документи є вибіркою з генеральної сукупності. Шукаючи незвичні треки за темпом ми використовували $stdDevPop, оскільки при розрахунку ми використовували всі треки, які є в колекції.

1. Що змінилося в плані виконання?
Спершу план виконання був COLLSCAN -> SORT і проходився по всім документам, відкидаючи ті, які не задовільняли умови фільтра. Після створення індекса план змінився на IXSCAN -> SORT -> FETCH. Відповідно скануються не всі документи, а одразу індекс, що і підтверджується даними нижче. Значення просканованих документів зменшується, а також з'явилися прочитані кліючі.

3. Чи це покривний запит?
```js
db.tracks.find({
  track_genre: "pop",
  popularity: { $gte: 70 }
});
```

Відповідь, ні. Покривним вважаєтсья запит, у якого:
1) всі поля фільтра містсяться в індексі;
2) всі поля, які повертаються в результаті, містяться в індексі;
3) _id має бути виключений з projection, якщо не міститься в індексі.

В прикладі вище поле `audio_features.danceability` відсутнє у фільтрі (вимога задовільняється частково, оскільки поля запиту присутні в індексі). Відсутній projection. І відповідно буде виведене поле _id, якого немає в індексі.

Отже, щоб даний запит був покривним, необхідно додати projection, який виключав би _id, а також описував результуючий документ полями, описаними в індексі (track_genre, audio_features.danceability, popularity):

```js
db.tracks.find(
  { track_genre: "pop", popularity: { $gte: 70 } },
  { _id: 0, track_genre: 1, popularity: 1, "audio_features.danceability": 1 }
);
```
